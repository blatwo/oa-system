# 瀚高工作记录系统 — 用户操作手册

| 版本 | V6.0 |
| 日期 | 2026-06-28 |

---

## 1. 快速开始

```bash
cd oa-system
python server.py          # http://localhost:8089

# 开发模式（HMR 热更新）
npx vite                  # http://localhost:3000
```

---

## 2. 模块操作指南

### 2.1 录入工作

- **AI 智能分类**：根据工作内容自动填充分类字段
- **AI 质疑** 🆕：点击"质疑"按钮，AI 检查工作内容的完整度（人名/职位、对象数量、操作细节等），弹出 Dialog 列出待补充项

### 2.2 工作记录列表

DataGrid 显示全部记录，支持日期/项目筛选、批量删除、JSON 导入导出。

### 2.3 项目工时统计

按项目分组汇总工时 + BSC 分类统计。

### 2.4 待办事项

- 项目下拉精确筛选
- 勾选完成/取消
- 项目待办 ≥5 条自动标「积压」

### 2.5 工作积压

自动分析："严重积压"/"有积压"/"正常"，进度条可视化。

### 2.6 加班记录

DataGrid 分页表格，字段：日期、签到、签退、工时、调休、报销、餐补、交通、工作内容。

### 2.7 AI 智能分析

选择模型 → 分析模式（项目总结/问题分析/自由提问）→ 开始分析，Markdown 渲染结果。

### 2.8 项目管理

#### 🆕 状态达成条件（V6.0 重点）
- **状态变更校验**：将项目状态改为"已完成"时，系统强制检查达成条件
- **项目级条件**：在"项目信息 → 编辑 → 达成条件 Tab"中维护
- **全局模板**：在"项目管理 → 状态达成条件"页面维护标准条件模板
- **应用模板**：一键将全局模板应用到指定项目

### 2.9 销售管理

手动添加关注的销售人员。Tab 2 自动统计每人负责的项目数。

### 2.10 用户画像

每个项目一份画像：商务面貌 / 技术面貌 / 痛点 / 关键人员 / 期望 / 备注。

### 2.11 产品规划

规划方案与产品目录三级分类关联。

### 2.12 版本发布

版本发布与知识产权（软著/专利/商标）关联。

### 2.13 兼容证明管理

按项目追踪兼容证明：应用系统、是否已做、当前环节（7 种状态）、开具/适配人、证明/适配数量、认领人。

### 2.14 工作模板 🆕

维护常用的工作记录模板（场景 + 内容 + 字典字段），录入时一键填充。

### 2.15 脚本模板 🆕

维护 SQL/shell 脚本模板，复制到剪贴板使用。

### 2.16 字典维护（子菜单）

- 常规字典 / 产品目录 / 服务目录 / 脚本模板

### 2.17 LLM 模型管理

配置 OpenAI 兼容 API 模型（测试连接 + 多模型切换）。

### 2.18 实用工具 🆕

包含 3 个 PostgreSQL 运维工具：

| 工具 | 用途 |
|------|------|
| **System ID 分析器** | 解析 PostgreSQL `system_identifier`（解码 initdb 时间戳、微秒、PID） |
| **LSN 分析器** | 解析 `pg_lsn`（XLogRecPtr 八字节结构：逻辑 WAL 文件号 / 偏移量） |
| **Patroni/etcd 调试** | Patroni 故障排查命令手册 + etcd 一致性检查步骤 |

## 3. 数据保护

每次启动 `python server.py` 自动备份 DB（保留最近 3 份）。备份文件：`oa_data.bak.YYYYMMDD_HHMMSS`。

---

## 4. 常见问题

| 问题 | 答案 |
|------|------|
| 项目下拉框没有？ | 先去"项目管理"新增 |
| 下拉选项不够？ | 字典维护添加 |
| 页面白屏？ | 确保 http://localhost:8089 |
| 换电脑迁移？ | 复制 oa_data.db |
| AI 分析失败？ | 检查 LLM 端点完整 + 测试连接 |
| 数据被删了？ | 用最近备份替换 oa_data.db |
| 项目状态改不了"已完成"？ | 检查"达成条件 Tab"中是否有未满足的条件 |
| 质疑功能没反应？ | 检查 LLM 模型是否配置并设为默认 |
