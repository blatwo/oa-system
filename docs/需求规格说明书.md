## 1. 引言

### 1.1 背景

瀚高数据库技术支持工程师邱臣君日常通过 Excel 管理多项目工作记录，涵盖项目跟踪、工时统计、BSC 任务管理、待办事项跟踪、加班记录、兼容证明等。随着项目数量增长（并行 10+ 项目），Excel 已无法满足效率要求。

### 1.2 目标

构建全栈工作记录管理系统，替代 Excel 完成日常工作录入、产品/服务管理、销售跟踪、用户画像、工时统计、待办跟踪、积压预警、加班管理、兼容证明追踪，以及基于大语言模型的智能分析。

### 1.3 范围

16 个功能模块：工作记录、待办事项、工作积压、加班记录、项目管理、销售管理（手动维护）、用户画像、产品目录、服务目录、产品规划、版本发布、兼容证明、字典维护、AI 智能分析、LLM 模型管理、DB 自动备份。

## 2. 功能需求

### FR-01 工作记录录入 / FR-02 工作记录列表 / FR-03 项目工时统计

（详见上版本，功能无变更。）

### FR-04 待办事项

- 项目下拉精确筛选
- 项目待办 ≥5 条时显示「积压」标签
- 勾选标记/取消完成

### FR-05 工作积压

| 评分维度 | 规则 |
|---------|------|
| 进行中数量 | ≥3 → +3分, ≥1 → +1分 |
| 未完成待办 | ≥5 → +3分, ≥2 → +1分 |
| 近两周工时 | ≥40h → +2分, ≥20h → +1分 |
| 最后活跃 | 超7天无活动且仍有进行中 → +2分 |

| 总分 | 状态 |
|------|------|
| ≥5 | 🔴 严重积压 |
| 3-4 | 🟠 有积压 |
| 0-2 | 🟢 正常 |

### FR-06 加班记录 🆕

| 字段 | 说明 |
|------|------|
| 日期 | YYYY-MM-DD |
| 签到/签退 | HH:MM |
| 加班工时 | 小时数（0.5 步长） |
| 调休 | 是/否 |
| 报销 | 是/否（绿色/灰色标签） |
| 餐补 | 金额 |
| 交通 | 金额 |
| 工作内容 | 简述加班做的事 |
| 备注 | 自由文字 |

- DataGrid 分页（15/25/50/100行可配），隔行换色，默认按日期降序
- 头部实时统计：累计工时 + 餐补合计 + 交通合计
- 已导入 117 条历史数据（2025-04-01 ~ 2026-05-17，360.5h）

### FR-07 项目管理

维护项目信息，供录入和统计使用。

### FR-08 销售管理（手动维护）🔄

- 按人统计负责项目数量

## 3. 数据保护 🛡️

> **DB 自动备份**：每次服务器启动，若 DB 已存在则自动备份为 `oa_data.bak.YYYYMMDD_HHMMSS`，保留最近 3 份。
> 
> **加表加列规则**：`CREATE TABLE IF NOT EXISTS` + `ALTER TABLE ADD COLUMN`，永不删库。
