## 1. 引言

### 1.1 背景

瀚高数据库技术支持工程师邱臣君日常通过 Excel 管理多项目工作记录，涵盖项目跟踪、工时统计、BSC 任务管理、待办事项跟踪、加班记录、兼容证明等。随着项目数量增长（并行 10+ 项目），Excel 已无法满足效率要求。

### 1.2 目标

构建全栈工作记录管理系统，替代 Excel 完成日常工作录入、产品/服务管理、销售跟踪、用户画像、工时统计、待办跟踪、积压预警、加班管理、兼容证明追踪、项目状态质量门、脚本库管理、PG 运维工具集，以及基于大语言模型的智能分析。

### 1.3 范围

V6.0 共 **22 个功能模块**：
1. 工作记录录入 2. 工作记录列表 3. 项目工时统计 4. 待办事项 5. 工作积压 6. 加班记录
7. 项目管理（含状态达成条件）8. 销售管理 9. 用户画像 10. 产品目录 11. 服务目录
12. 产品规划 13. 版本发布 14. 兼容证明 15. 字典维护 16. 优先级准则 17. 典型案例库
18. LLM 模型管理 19. AI 智能分析 20. 工作模板 21. 脚本模板 22. 实用工具集

## 2. 功能需求

### FR-01 工作记录录入

- 两列网格布局，含 AI 智能分类
- 🆕 **AI 质疑**：检测工作内容完整度（人名/职位、对象数量、操作细节等），返回待补充项列表

### FR-02 工作记录列表
DataGrid 表格，日期/项目筛选，批量删除，JSON 导入导出。

### FR-03 项目工时统计
按项目分组汇总工时 + BSC 分类统计。

### FR-04 待办事项
项目下拉精确筛选，项目待办 ≥5 条时显示「积压」标签。

### FR-05 工作积压

| 评分维度 | 规则 |
|---------|------|
| 进行中数量 | ≥3 → +3分, ≥1 → +1分 |
| 未完成待办 | ≥5 → +3分, ≥2 → +1分 |
| 近两周工时 | ≥40h → +2分, ≥20h → +1分 |
| 最后活跃 | 超7天无活动且仍有进行中 → +2分 |

| 总分 | 状态 |
|------|------|
| ≥5 | 🔴 严重积压 |
| 3-4 | 🟠 有积压 |
| 0-2 | 🟢 正常 |

### FR-06 加班记录
DataGrid 分页，字段：日期、签到、签退、工时、调休、报销、餐补、交通、内容、备注。

### FR-07 项目管理（含状态达成条件）🆕

- 项目信息 CRUD（编码自动生成 HG-YYYY-NNN）
- 🆕 **状态达成条件**：
  - 全局条件模板（project_id=0，可应用到项目）
  - 项目级条件独立管理
  - 状态→"已完成"时**强制校验条件**（不满足返回 400）
  - 操作审计日志（project_status_condition_logs）

### FR-08 销售管理（手动维护）
无种子数据，手动添加关心的销售人员，按人统计负责项目数量。

### FR-09 用户画像
每个项目一份画像：商务面貌/技术面貌/痛点/关键人员/期望/备注。

### FR-10 产品目录 / FR-11 服务目录
三级树形层级结构，CRUD。

### FR-12 产品规划 / FR-13 版本发布
产品规划关联目录，版本发布关联知识产权。

### FR-14 兼容证明
7 种状态流转（待启动/适配中/已适配/开具中/已开具/延期/取消）。

### FR-15 字典维护
Tab 形式管理分类和字典项。

### FR-16 优先级准则
重要/不重要/紧急/不紧急四象限。

### FR-17 典型案例库
产品/场景/影响/发现人/服务分类。

### FR-18 LLM 模型管理
OpenAI 兼容 API 多模型配置（测试连接 + 默认切换）。

### FR-19 AI 智能分析
三种模式：项目总结、问题分析、自由提问，Markdown 渲染。

### FR-20 工作模板 🆕
常用工作记录模板，录入时一键填充。

### FR-21 脚本模板 🆕
SQL/shell 脚本模板，复制到剪贴板。

### FR-22 实用工具 🆕

| 工具 | 说明 |
|------|------|
| System ID 分析器 | 解析 PostgreSQL `system_identifier`（解码时间戳/微秒/PID） |
| LSN 分析器 | 解析 `pg_lsn`（逻辑 WAL 文件号 / 偏移量） |
| Patroni/etcd 调试 | 故障排查命令手册 + etcd 一致性检查 |

## 3. 数据保护 🛡️

> **DB 自动备份**：每次服务器启动自动备份 `oa_data.bak.YYYYMMDD_HHMMSS`，保留最近 3 份。
>
> **加表加列规则**：`CREATE TABLE IF NOT EXISTS` + `ALTER TABLE ADD COLUMN`，永不删库。

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 页面加载 < 3s，API 响应 < 500ms |
